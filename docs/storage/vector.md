# Storage — Vector Store

The vector store enables semantic queries against bill content — finding bills by meaning, not just keyword or tag.

## Design

**Embedding target:** the `summary` + concatenated `key_clauses[].impact` text from `analysis.json`. This is 2–5 sentences of dense, human-readable meaning — a better embedding target than raw legal text.

**Storage options:**

| Option | When to use |
|--------|-------------|
| `sqlite-vec` | MVP and solo use — zero extra services, embeds alongside `stateconscious.db` |
| `chromadb` | Small team or larger corpus — file-based, no server required |
| `pgvector` (PostgreSQL) | Multi-user / production scale |

**MVP default:** `sqlite-vec`. It keeps the full system in a single SQLite file with no external dependencies.

## Embedding pipeline

```
analysis.json
  → build embedding text (summary + clause impacts)
  → call embedding model (e.g. text-embedding-3-small)
  → store vector alongside bill_analysis row
```

The embedding model is configured in `.env` alongside the LLM.

## Query pattern

```python
# Semantic search: "employment termination rights"
results = vector_store.search(
    query="employee termination rights during probation",
    filter={"country": "my", "year": 2024},
    top_k=10
)
# Returns: list of (bill_sha256, similarity_score)
# Join with bill_analysis for full metadata
```

## Planned schema addition

A `bill_vector` table (or `sqlite-vec` virtual table) with:

| Column | Description |
|--------|-------------|
| `sha256` | FK → `bill_analysis.sha256` |
| `embedding` | float[] vector (dimension = model-dependent) |
| `embed_model` | Model ID used to generate |
| `embed_text` | The text that was embedded (for re-embed auditing) |
| `embedded_at` | ISO-8601 |

## Re-embedding

When a better embedding model becomes available, existing vectors can be regenerated by querying `bill_vector` for rows with the old `embed_model` and re-running the pipeline with `--force`.

## What this enables

- "Show me all bills that could affect my construction business" (semantic similarity)
- "Find bills similar to the Personal Data Protection Act amendments" (bill-to-bill similarity)
- "What changed in employment law last year" (semantic + year filter)
- Cross-country comparisons once multiple source adapters are added
